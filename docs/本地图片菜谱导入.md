# 本地图片菜谱导入

图片菜谱可能包含私人内容，因此识别结果不会写入 Git 仓库，也不会上传至任何第三方服务。`scripts/extract-sui-one-image-recipes.py` 只在本机运行，输出两类私有文件：

- `markdown/*.md`：每道菜一份可阅读、可手工修订的菜谱文档，保留 OCR 原文和图片相对路径。
- `sui-one-recipes.json`：供快乐厨房导入的结构化数据。这个文件应放在 NAS 的持久化数据目录，不能提交到公开仓库。

## 本机识别

安装本机依赖后，以源目录和一个位于源目录外或受访问控制的导出目录运行脚本：

```powershell
python -m pip install rapidocr_onnxruntime pillow numpy
python scripts/extract-sui-one-image-recipes.py `
  --input 'E:\菜谱图片' `
  --output 'E:\菜谱图片\快乐厨房导出'
```

大量图片适合分批执行：每批使用 `--offset` 与 `--limit` 并加上 `--fragment`。所有批次完成后，运行一次 `--merge-fragments`，即可生成最终的 `sui-one-recipes.json` 和 `import-report.json`。

```powershell
python scripts/extract-sui-one-image-recipes.py `
  --input 'E:\菜谱图片' `
  --output 'E:\菜谱图片\快乐厨房导出' `
  --merge-fragments
```

## 导入 NAS 中的家庭

将私有 JSON 复制到 Docker Compose 配置的 `APPDATA_PATH` 下：

```text
/mnt/user/appdata/happy-kitchen-stack/data/imports/sui-one-recipes.json
```

容器中的对应路径为 `/data/imports/sui-one-recipes.json`。登录目标家庭的成人账号后，打开 **菜谱 → 我的菜谱**，点击 **导入本地菜谱**。

若本机 Docker Desktop 使用示例默认值 `APPDATA_PATH=./data`，则对应的宿主机路径是项目目录下的 `data/imports/sui-one-recipes.json`。

导入严格归属当前登录账号所在的家庭，并通过图片来源目录去重，重复点击不会再次创建同一批菜谱。OCR 置信度不足、食材未结构化或存在待确认文本的菜谱会标记为“数据待补”，不会参与自动推荐和周食谱；请先在“我的菜谱”中补充/核对后再使用。
